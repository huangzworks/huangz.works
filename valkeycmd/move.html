MOVE

MOVE命令可以将键从客户端当前使用的数据库（源数据库）移动到指定的数据库（目标数据库）中：

MOVE key db

命令在成功移动键时返回1，由于键不存在于源数据库等原因而导致移动失败时返回0。

以下代码展示了如何使用MOVE命令，将msg键从当前正在使用的0号数据库移动到指定的9号数据库：

127.0.0.1:6379> SET msg "hello from db 0"
OK
127.0.0.1:6379> MOVE msg 9
(integer) 1

使用SELECT命令和GET命令可以确认此次移动操作已经成功：

127.0.0.1:6379> SELECT 9
OK
127.0.0.1:6379[9]> GET msg
"hello from db 0"

覆盖已有的同名键

在默认情况下，MOVE命令在尝试将键从源数据库移动到目标数据库之前，会先检测同名键在目标数据库中是否存在，如果存在则放弃执行移动操作，并返回0表示移动失败。

但是通过Valkey 10.0新增的REPLACE可选项，MOVE命令可以在同名键已经存在于目标数据库的情况下，直接使用被移动的键去覆盖已有的键：

MOVE key db [REPLACE]

举个例子，在执行上一节展示的命令代码之后，msg键已经存在于9号数据库，如果这时再在0号数据库创建一个msg键并尝试使用MOVE将其移动至9号数据库，那么此次移动将会失败：

127.0.0.1:6379> SET msg "HELLO AGAIN from db 0"
OK
127.0.0.1:6379> MOVE msg 9
(integer) 0

但是只要加上REPLACE可选项，MOVE命令就会成功执行，并使用新设置的msg键覆盖9号数据库中已有的同名键：

127.0.0.1:6379> MOVE msg 9 REPLACE
(integer) 1
127.0.0.1:6379> SELECT 9
OK
127.0.0.1:6379[9]> GET msg
"HELLO AGAIN from db 0"

其他信息

MOVE命令从Valkey 1.0版本开始可用，并从10.0版本开始支持REPLACE可选项；命令的复杂度为O(1)。
